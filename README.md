## Описание приложения

«Запись на звонок» — упрощенный сервис бронирования времени для встреч по мотивам Cal.com.
У приложения один основной сценарий: владелец публикует доступное время для встреч, а гость выбирает свободный слот и записывается на звонок.

Ограничения намеренные:
- нет регистрации и авторизации
- нет личных кабинетов
- нет интеграций с внешними календарями

Роли в системе:
- Владелец календаря: один заранее заданный профиль (единственный владелец)
- Гость: бронирует слоты без аккаунта и без входа в систему

## Доменные сущности

### Владелец (Owner)

Единственный профиль владельца календаря, который управляет типами событий и просматривает будущие встречи.
Авторизация отсутствует, поэтому владелец фиксированный и заранее задан.

### Тип события (Event Type)

Шаблон встречи, который выбирает гость при бронировании.
Поля:
- `id`: уникальный идентификатор
- `title`: название
- `description`: описание
- `durationMinutes`: длительность в минутах

Примечание: длительность задается на уровне типа события. При выборе типа события гостю показываются слоты с такой же длительностью.

### Слот (Slot)

Интервал доступности, который владелец создает вручную через UI, чтобы гость мог выбрать время для звонка.
Слот характеризуется:
- временем начала (`startAt`)
- длительностью в минутах (`durationMinutes`)
- временем окончания (`endAt`), вычисляется как `startAt + durationMinutes`

Слоты создаются и показываются в пределах доступности владельца: с 09:00 до 18:00 (по времени приложения/владельца).

### Бронирование (Booking)

Факт записи гостя на конкретный слот и конкретный тип события.
Бронирование связывает:
- выбранный `eventTypeId`
- выбранный слот (его интервал: `startAt`, `endAt`)

Связь слота и типа события: слот не привязан к типу события, но для выбранного типа события гостю показываются только слоты с совпадающей длительностью (`slot.durationMinutes === eventType.durationMinutes`).

Дополнительно может хранить контактные данные гостя (например, имя и email), если это потребуется в реализации.

## Публичный сценарий гостя

1. Гость открывает страницу с доступными типами событий и видит для каждого типа: название, описание, длительность.
2. Гость выбирает тип события, переходит к календарю/расписанию и видит свободные слоты подходящей длительности в окне 09:00–18:00.
3. Гость выбирает слот и создает бронирование.

## Правило занятости (конфликт бронирований)

Нельзя создать бронирование, если его временной интервал пересекается с интервалом любого существующего бронирования, даже если выбраны разные типы событий.

Рекомендуемая формализация интервалов: `[startAt, endAt)`. Тогда бронирования, которые стыкуются (одно заканчивается ровно в момент начала другого), не считаются конфликтом.

## Страница владельца: предстоящие встречи

Владелец открывает страницу предстоящих встреч и видит единый список бронирований по всем типам событий.
